# CONTENT-GUIDE.md — コンテンツ編集ルール

## ブログ記事を追加する

### 方法A: Decap CMS（推奨・GUI）

1. ブラウザで `https://[Netlifyサイト]/admin/` にアクセス
2. **「Login with GitHub」** をクリック
3. GitHub の認可画面で **「Authorize」**
4. 「ブログ記事」コレクションを開く → **「+ New ブログ記事」**
5. 入力:
   - **タイトル**: 記事タイトル
   - **投稿日**: YYYY-MM-DD 形式
   - **一覧表示用の説明文**: 120文字以内（SEO・SNSシェア用）
   - **サムネイル画像**: アップロード（`images/blog/` に保存される）
   - **本文**: Markdown で記述
6. 右上の **「Publish → Publish now」**
7. 自動で GitHub にコミット → Netlify がビルド → 公開（通常 1〜2分）

### 方法B: 直接 commit（手動）

1. `_posts/YYYY-MM-DD-スラッグ.md` を作成
2. ファイル内容のテンプレート:

```markdown
---
title: 記事タイトル
date: 2026-05-22
description: 一覧表示用の説明文（120文字以内）
thumbnail: /images/blog/サムネイル画像.jpeg
---

本文を Markdown で記述します。

![画像の説明](/images/blog/画像ファイル名.jpeg)

## 見出し2

普通の段落。

### 見出し3

- 箇条書き
- 箇条書き
```

3. 画像は `images/blog/` 配下に配置
4. `git add . && git commit -m "Add blog post: タイトル" && git push`

**重要な制約**:
- ファイル名は `YYYY-MM-DD-スラッグ.md` 形式（日付がないと Jekyll が無視）
- ファイル名と front matter の `date` を一致させる
- ファイル名のスラッグは半角英数推奨（日本語も動くがURLが見にくい）
- `layout: post` は `_config.yml` の `defaults` で自動付与されるので不要

## 固定ページを編集する

各 HTML ファイルを直接編集 → `git push`：

| 編集対象 | ファイル |
|---------|---------|
| トップページ | `index.html` |
| プロフィール | `profile.html` |
| 政策 | `policy.html` |
| 議会活動 | `activity.html` |
| ブログ一覧 | `blog.html` |
| 市政報告レポート | `newsletter.html` |
| お問い合わせ | `contact.html` |
| ご支援のお願い | `support.html` |
| 一般質問記録 | `qa-rX-Y.html`（例: `qa-r8-3.html` = 令和8年3月） |

### スタイル変更

- 全体: `css/style.css`
- スクリプト（メニュー・スライダー等）: `js/main.js`
- ブログ記事のレイアウト: `_layouts/post.html`

## 一般質問記録の追加

各定例会ごとに `qa-rX-Y.html` を追加するスタイル：

- 命名規則: `qa-r{令和年}-{月}.html`
  - 例: 令和8年3月定例会 → `qa-r8-3.html`
- 既存ファイルをコピーして内容を差し替えるのが早い
- 追加後は `activity.html` のリンク一覧にも追記すること

## 画像の取り扱い

### 配置場所

| 用途 | フォルダ |
|------|---------|
| サイト全体（ヒーロー・プロフィール等） | `images/` |
| ブログ記事内画像 | `images/blog/` |

### 推奨フォーマット

- 写真: JPEG（`.jpg`/`.jpeg`）
- 図解・スクリーンショット: PNG
- HEIC は使わない（古い JPG 化版を残してあるが iOS デフォルトの HEIC は表示できないブラウザがある）

### サイズの目安

- ヒーロー画像: 横 1600px 以上（高解像度ディスプレイ対応）
- ブログサムネイル: 800×600px 程度（一覧では 240×160 に縮小表示）
- 本文中画像: 横 800〜1200px

### 命名規則

- 半角英数推奨
- スペースは使わずアンダースコアまたはハイフン
- 日本語ファイル名は CMS で文字化けする可能性あり

## SEO・SNS シェア対策

### 各記事に必須

- `title`: 検索結果に表示される（front matter）
- `description`: 一覧・SNSシェア時に表示される（120文字以内）
- `thumbnail`: SNSシェア・一覧サムネイル

### サイト全体で未対応

- **OGP（Open Graph Protocol）タグ**: 現状未実装
  - 実装するなら各 HTML の `<head>` に追加
  - 例: `<meta property="og:title" content="...">`
- **Twitter Card タグ**: 同上
- **Favicon**: 現状未実装
- **sitemap.xml**: 存在するが手動更新（Jekyll プラグイン jekyll-sitemap の導入を検討）

## 文体・トーンの指針

メインサイトの記事は CLAUDE.md 上位（`C:\Users\gikai044\プロジェクト\議員活動担当\SNS担当\CLAUDE.md`）に詳細な文体ガイドあり。要点：

- **です・ます調**が基本
- **Q&A 構造**: 「〜はどうなるの？」「〜できるの？」と問いを立てて行替えで解答
- **個人の成果を明示**: 「私も議会質問で〜求めました」
- **数字・データ主導**: 予算額・人数・完成予定年度を多用
- **令和・西暦の併記**: 「令和11年度（2029年度）」
- **地域名の具体化**: 相武台・麻溝台・新磯野・相模台・当麻・田名など
- **読者への呼びかけ**: 「皆様のご意見・ご要望を承ります」
